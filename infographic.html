Section 02   五個工具設計
T1
search_papers
搜尋 arXiv 論文
IN: query, max_results OUT: papers list
T2
summarize_paper
產生繁體中文結構化摘要
IN: title, abstract OUT: structured summary
T3
save_to_knowledge_base
存入本地 ChromaDB 知識庫
IN: title, abstract, arxiv_id, url, tags, year OUT: confirmation + count
T4
query_knowledge_base
兩層 RAG 查詢，回傳信心度指標
IN: question, use_chunks OUT: papers + chunks + kb_sufficient
T5
expand_context
擴展段落上下文（前後相鄰 chunk）
IN: arxiv_id, chunk_index OUT: expanded text
Section 03   ChromaDB 兩層記憶架構
Layer 1 — 摘要層
paper_abstracts Collection
粒度：論文整篇（每篇一筆）
嵌入：title + abstract 全文
Metadata：title, arxiv_id, url, tags, year
模型：paraphrase-multilingual-MiniLM-L12-v2
⚡ 用途：Confidence Router 快速判斷知識庫是否充足
閾值：distance < 0.7 → kb_sufficient = true
↕ 同一篇論文
Layer 2 — 區塊層
paper_chunks Collection
粒度：300 字 chunk，overlap 50 字
Metadata：arxiv_id, chunk_index, total_chunks
查詢：top-5 相似段落
🔬 用途：細粒度 RAG，精準定位論文中的具體段落
expand_context 可取前後 chunk 補完上下文

設計參考 & 架構說明
// DESIGN DECISIONS
兩層 RAG 架構 參考 local-ai-knowledge-agent 的 hierarchical indexing 設計，Layer1 快速路由、Layer2 精準檢索
信心度路由 以 ChromaDB 餘弦距離作為信心指標，distance < 0.7 視為知識庫已有足夠資料，避免重複搜尋
多語言嵌入 使用 paraphrase-multilingual-MiniLM-L12-v2，支援中文查詢對應英文論文內容
Chainlit 視覺化 每個 Tool 呼叫以 cl.Step 即時顯示，讓 AI 決策過程完全透明可觀察
Function Calling 使用 Gemini protos.FunctionDeclaration 原生機制，LLM 自主決策工具呼叫順序與參數
